# 구현 계획 · AX해커톤 예선 2차 제출(기업 미정)

> 처음부터 전체를 달리지 않는다. 가장 작은 성공 단위부터. 각 단계는 완료 조건과 검증 방법을 명시한다.

## 단계 개요
| 단계 | 목표 | 완료 조건 | 검증 방법 | 의존/연결 |
|------|------|-----------|-----------|-----------|
| S1 | 기업 선정 | 후보 1곳 확정 + Decisionlog 기록 | `docs/company-selection.md` 체크리스트 완료 | 없음 |
| S2 | 문제 리서치 심화 | 확정 기업의 문제를 공개 자료 3건 이상으로 뒷받침 | 출처(URL/제목/확인일) 기록 확인 | S1 |
| S3 | 플러그인 설계 | 문제 → 해결 방식 → 필요한 스킬/코드/MCP 구성 확정 | `docs/requirements-contract.md` 갱신 완료 | S2 |
| S4 | 플러그인 구현 | `src/.codex-plugin/plugin.json` + 스킬/코드 동작 | 로컬에서 Codex 플러그인 실행 확인 | S3 |
| S5 | 검증 및 질문 5문항 작성 | `docs/validation-plan.md` 항목 통과, 질문 5문항 답변 완료 | 문서-코드-로그 정합성 점검 | S4 |
| S6 | 패키징·제출 준비 | `submission.zip` 구조 요건 충족 | 구조 대조 체크리스트 | S5 |

## 단계 상세

### S1 · 기업 선정 (현재 단계)
- 왜 필요한가: 과제 규정상 기업마다 별도 제출이 필요하고, 근거가 공개 자료여야 하므로 잘못 고르면 이후 전체 작업이 무효화될 위험이 큼.
- 이전 단계와 연결: 카카오페이증권 제출 완료 상태에서 이어짐.
- 다음 단계로 전달: 확정된 기업명과 1차 문제 후보.
- 완료 조건: `docs/company-selection.md`의 후보 5곳 조사·채점 완료, 1곳 확정, `Decisionlog.md`에 다음 Decision ID로 기록.
- 검증 방법: 선정된 기업의 문제 근거가 공개 자료 링크로 확인되는지 재확인.
- 실패 시 중단점: 근거 확보 가능성이 모든 후보에서 낮게 나오면(기준 1에서 전부 탈락) 사용자에게 보고하고 후보 재검토.

### S2~S6
- 기업 확정 전에는 세부를 채우지 않는다(추측 방지). S1 완료 후 이 문서를 갱신한다.
