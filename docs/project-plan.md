# 프로젝트 기획 · AX해커톤 예선 2차 제출(무신사 문제 2)

## 문제 정의
- `docs/과제.md` 과제: 선택한 기업(또는 그 산업·고객)이 겪는, 공개 자료로 입증 가능한 실제 문제를 해결하는 Codex 플러그인을 만든다.
- 확정 기업·문제: **무신사 / 문제 2 · 상품 데이터 에이전트화 변환기**.
- 문제: 무신사의 상품 상세정보는 소비자에게는 읽히지만, AI 에이전트가 일관되게 질의·필터·설명하기에는 비정형 텍스트 중심이다. 무신사 테크리드 인터뷰의 “에이전트가 사용하는 첫 번째 도구가 무신사이길”이라는 방향과, 여러 플랫폼·상품 데이터 파편화 문제에 대응하려면 에이전트가 이해할 구조화 상품 데이터가 필요하다.

## 목표와 성공 기준
- 상품 상세페이지 텍스트(BYO 입력)를 정규화된 JSON으로 변환한다.
- 아우터·상의 2개 카테고리에 대해 카테고리, 소재, 핏·실루엣, 컬러, 계절, TPO·스타일 태그, 관리, 사이즈 등 핵심 속성을 추출·표준화한다.
- 같은 상품의 다른 표현을 속성 유사도로 중복 후보로 감지한다.
- 변환된 JSON을 바탕으로 자연어 패션 질의(예: “여름 하객룩”)에 대응할 수 있는 descriptor를 만든다.
- `submission.zip` 구조(`src/`, `README.md`, `logs/`) 요건을 충족하고, 질문 5문항에 로그·플러그인과 정합성 있게 답변한다.
- 마감(2026-07-06 밤) 전 구현·검증·패키징을 완료한다.

## 범위
- 포함: 무신사 문제 2 근거 정리, 기준 계약 문서 고정, Codex 플러그인 설계·구현, 더미 픽스처 검증, 로컬 Codex CLI 실행 시연, 로그 정리, 질문 5문항 작성, 제출 패키징.
- 제외: 카카오페이증권 관련 작업(별도 제출 완료), 무신사 문제 1(상품정보 표기 검수기), 여러 기업을 하나의 제출물로 묶는 것, 제3자 사이트 자동 크롤링, 내부/비공개 데이터 사용.

## 주요 사용자와 사용 시나리오
- 해커톤 심사위원/주최측이 `submission.zip`을 열어 `plugin.json`·스킬·스크립트·로그·README·질문 답변의 정합성을 확인하며 평가한다.
- 시연 사용자는 상품 상세 텍스트를 직접 붙여넣고, 플러그인은 구조화 JSON과 부족·모호 속성, 중복 후보, 에이전트 질의 descriptor를 반환한다.
