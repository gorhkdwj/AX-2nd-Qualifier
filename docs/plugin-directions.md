# 후보 3사 · 문제 정의와 플러그인 방향

> 작성일: 2026-07-04 KST
> 전제: 채널톡·삼일PwC는 후보에서 제외. 남은 3사(무신사 → 메디테라피 → 마이리얼트립)에 대해, **외부 공개 자료만으로도** 해결·개선·제안이 가능한 문제를 각 3개씩 정리하고 Codex 플러그인 방향을 제시합니다.
> 근거: `docs/company-research-final-report.md`에서 `[검증됨]`·`[교차확인]`으로 확인된 공개 사실만 사용합니다. 미검증 추정은 문제로 세우지 않습니다.
> **선정 자체는 하지 않습니다.** 각 사의 실행 후보를 정리한 참고 문서입니다.

## 공통 원칙 — "공개 자료만으로"가 왜 중요한가
과제 규정상 문제 근거는 공개·검증 가능해야 하고, 플러그인은 심사자가 재현·검증할 수 있어야 합니다. 따라서 아래 문제·플러그인은 모두 **기업 내부 비공개 데이터 없이**, 다음과 같은 공개 소스만으로 동작·시연하도록 설계했습니다.
- 공개 상품 상세페이지·랜딩페이지·공식 도움말/정책 문서
- 공개 법령·행정 고시(전자상거래 상품정보제공고시, 화장품 표시·광고 지침, 섬유 표시기준 등)
- 공개 리뷰·Q&A·뉴스·공시

각 플러그인의 형태는 Codex 플러그인 표준(`src/.codex-plugin/plugin.json` + `skills/<이름>/SKILL.md`, 필요 시 실행 코드·MCP)을 따릅니다.

## 1. 무신사

무신사에서 공개자료로 검증된 핵심 문제는 **상품 정보(특히 소재 혼용률) 표기 오류의 반복**과 그로 인한 **표시 규정 준수·평판 리스크**입니다. 상품 상세페이지·표기 기준·리뷰가 모두 공개라 플러그인화에 유리합니다.

### 문제 1-A · 소재 혼용률·케어라벨 표기 정합성 오류 (반복 발생)
- **근거(공개)**: 2024.12~2025.1 다운·캐시미어 7,968개 전수조사에서 42개 브랜드·165개 상품 오기재 적발(무신사 뉴스룸 2025-0123-2, 이투데이) → 2025.11~12 노스페이스 패딩 13개 스타일(28 SKU) 혼용률 오기 재발, 대행사 벌점 50점·선제 전액 환불(연합뉴스TV·무신사 뉴스룸 2025-1202-2·스카이데일리·산경투데이). 재발 방지책에도 1년 내 동일 유형 재발 `[검증됨/교차확인]`.
- **공개자료만으로 가능한 이유**: 점검 대상인 상품 상세페이지의 표기(혼용률 수치·소재명·케어라벨 이미지)와 판정 기준(무신사가 도입한 "기타 섬유" 포괄표기 금지·15% 이내 세분화 표기, 국가기술표준원 섬유 표시기준)이 모두 공개. 내부 검수 시스템 접근이 불필요.
- **플러그인 방향 — `무신사 표기 정합성 점검기`**
  - 입력: 상품 상세페이지 텍스트(붙여넣기/URL) + (선택) 케어라벨 이미지
  - 처리: ① 혼용률 합계 100% 여부·논리 오류 검출 ② "기타 섬유" 등 포괄표기 사용 여부 ③ 15% 이내 세분화 규칙 위반 ④ 텍스트 표기 ↔ 케어라벨 이미지(OCR) 불일치 ⑤ 소재명 표준용어 매핑
  - 출력: 위반 항목·근거 규정 인용·수정 제안·승인/보류 판정
  - 형태: SKILL(판정 규칙·프롬프트) + 표기 규칙/표준용어 사전(정적 데이터) + (선택) 이미지 OCR 코드

### 문제 1-B · 상품정보제공고시 필수항목 누락·불일치
- **근거(공개)**: 위 혼용률 사태의 본질은 "온라인 상세페이지 상품정보의 신뢰성"이며, 전자상거래상 **상품정보제공고시**(의류: 소재·제조국·제조자·세탁방법·치수·A/S 등 필수 표시항목)는 공개 규정. 표기 오류가 반복된다는 사실이 검증됨 `[검증됨]`.
- **공개자료만으로 가능한 이유**: 필수 표시항목 목록(공정위/한국소비자원 고시)과 점검 대상 상세페이지가 모두 공개.
- **플러그인 방향 — `상품정보 고시항목 검수기`**
  - 입력: 상품 상세페이지(텍스트/URL), 카테고리
  - 처리: 카테고리별 상품정보제공고시 필수항목 체크리스트 대조 → 누락·공란·상호 모순(예: 소재 표기와 세탁법 불일치) 검출
  - 출력: 누락 항목·근거 고시 조항·보완 문구 초안
  - 형태: SKILL + 카테고리별 필수항목 데이터(정적)

### 문제 1-C · 공개 리뷰·뉴스 기반 품질·평판 리스크 조기경보
- **근거(공개)**: 표기 오류 외에도 오프라인 응대 논란(2026.3, 경향신문), 과거 광고 문구 재조명·공식 사과(2026.5, ZDNet·무신사 뉴스룸 2026-0520-02) 등 평판 이슈가 반복 `[교차확인]`. 반복성이 문제.
- **공개자료만으로 가능한 이유**: 상품 리뷰·Q&A·뉴스·커뮤니티 글이 공개. 특정 상품/이슈의 부정 신호 급증을 공개 데이터만으로 관측 가능.
- **플러그인 방향 — `평판·품질 신호 조기경보기`**
  - 입력: 상품/브랜드명 또는 공개 리뷰·뉴스 텍스트 모음
  - 처리: 소재·불량·오배송·표기·응대 등 리스크 카테고리 분류 → 부정 신호 빈도·급증 여부 요약 → 선제 대응이 필요한 상위 항목 도출
  - 출력: 리스크 카테고리별 요약·대표 인용·대응 우선순위
  - 형태: SKILL(분류·요약 규칙). 수집은 사용자가 공개 리뷰/뉴스를 입력하는 방식으로 한정(크롤링 의존 최소화)

## 2. 메디테라피

메디테라피에서 공개자료로 검증된 문제는 **퍼포먼스 광고 대비 낮은 구매전환/ROAS·진정성 콘텐츠 부족**입니다. 뷰티 D2C 특성상 **화장품 표시·광고 규정**과 **공개 리뷰**가 강력한 공개 근거가 됩니다. (부작용·품질 의혹은 메디테라피 특정 공개 근거가 없어 문제로 세우지 않습니다.)

### 문제 2-A · 화장품 표시·광고 규정 위반 리스크 (과대광고 등)
- **근거(공개)**: 고효능 성분 브랜드로서 광고 문구의 규제 리스크가 큼. 식약처 **화장품 표시·광고 관리 지침**(민원인안내서, 2025.8 게시)이 공개 규정으로 존재 `[검증됨]`. 자사몰 상세·광고 문구도 공개.
- **공개자료만으로 가능한 이유**: 판정 기준(식약처 지침·화장품법상 금지표현)과 점검 대상(공개 광고·상세 문구)이 모두 공개.
- **플러그인 방향 — `화장품 광고문구 리스크 점검기`**
  - 입력: 광고/상세페이지 문구, 제품 유형(일반/기능성)
  - 처리: 의약품 오인·질병 치료 암시·배타적/최상급 표현·미승인 효능 표방 등 금지 표현을 식약처 지침 기준으로 탐지 → 위험도 등급화
  - 출력: 위반 소지 문구·근거 지침 조항·대체 표현 제안
  - 형태: SKILL + 금지/주의 표현 규칙 사전(정적). 뷰티 브랜드 대상 강력한 실증 가능

### 문제 2-B · 퍼포먼스 광고 → 구매전환 병목 진단 및 콘텐츠 제안
- **근거(공개)**: 피처링 고객사례 인터뷰에 "광고 유입이 구매로 이어지지 못해 ROAS 개선 돌파구가 절실", "진정성 있는 브랜드 콘텐츠 부족"이 담당자 발언으로 확인됨 `[검증됨]`.
- **공개자료만으로 가능한 이유**: 랜딩/상세페이지 문구·제품 USP·공개 리뷰가 모두 공개. 실제 캠페인 성과 데이터(비공개) 없이도 "메시지·구조" 관점의 진단·제안이 가능(MVP는 공개 페이지로 시연).
- **플러그인 방향 — `전환 병목 진단 & 콘텐츠 코치`**
  - 입력: 랜딩/상세페이지 문구, 제품 USP, (선택) 공개 리뷰
  - 처리: 전환 관점 체크(가치제안 명확성·신뢰요소·CTA·구매장벽) → 병목 진단 → 공개 리뷰에서 실제 고객 언어를 뽑아 "진정성 콘텐츠" 초안 제안
  - 출력: 병목 진단표·개선 우선순위·리뷰 기반 카피 초안
  - 형태: SKILL. (선택) 리뷰 요약 코드

### 문제 2-C · 다국어 공개 리뷰 이슈·감성 분석
- **근거(공개)**: 대만·일본·아마존 북미·쇼피 동남아 등 글로벌 다채널 확장 확인(사람인 연혁) `[검증됨]`. 각 채널의 리뷰가 공개.
- **공개자료만으로 가능한 이유**: 아마존·쇼피·올리브영 등 공개 리뷰만으로 이슈를 관측·분류. (특정 부작용을 단정하지 않고, "리뷰에 실제로 나타난 내용"만 근거로 사용 — 검증 원칙 준수.)
- **플러그인 방향 — `글로벌 리뷰 이슈 트래커`**
  - 입력: 공개 다국어 리뷰 텍스트 모음(채널별)
  - 처리: 번역·정규화 → 효능/사용감/자극/배송/디바이스 등 카테고리 분류 → 채널·언어별 부정 이슈 비중·대표 인용 요약
  - 출력: 이슈 카테고리 리포트·급증 신호·대응 제안
  - 형태: SKILL + (선택) 번역/집계 코드. 수집은 사용자가 공개 리뷰를 입력하는 방식으로 한정

## 3. 마이리얼트립

마이리얼트립에서 공개자료로 검증된 문제는 **예약·취소·환불 문의의 구조적 복잡성**과 **공개 도움말/정책 콘텐츠 자체의 정합성**입니다. 공식 도움말·정책 문서가 전부 공개라, "구조 개선·제안"형 플러그인에 특히 적합합니다.

### 문제 3-A · 예약/취소/환불 문의의 분류·정확한 규정 안내 부담
- **근거(공개)**: 도움말이 예약 33개·취소환불 7개·상품문의 26개로 세분되어 있고 24시간 채팅 상담 운영(help.myrealtrip.com) `[검증됨]`. 문의 유형이 많고 규정이 복잡함이 공개 구조로 드러남.
- **공개자료만으로 가능한 이유**: 답변 근거가 되는 공식 도움말 전체가 공개 → 이를 지식원으로 삼는 RAG(문서 검색형 답변)로 충분.
- **플러그인 방향 — `여행 문의 triage & 답변 초안기`**
  - 입력: 고객 문의 문장(+선택: 예약 유형/시점)
  - 처리: 문의 유형 분류 → 관련 공식 도움말 검색·인용 → 답변 초안 + 추가 확인 정보 목록 생성
  - 출력: 문의 유형·근거 FAQ 링크·답변 초안·필요 확인정보
  - 형태: SKILL + 공식 도움말 코퍼스(공개 문서 수집) + 검색 로직

### 문제 3-B · 취소·환불 규정 복잡성(항공사별·사유별)로 인한 오안내
- **근거(공개)**: 항공사별 24시간 무료취소 정책 문서, "부득이한 사유" 취소 가이드(증빙 필요·영업일 7일 이내 안내), 예약확정 SLA(영업일 2일) 등이 모두 공개 도움말로 존재 `[검증됨]`. 규정이 조건별로 갈려 오안내 위험이 큼.
- **공개자료만으로 가능한 이유**: 적용 규정·필요 증빙·예상 소요일 산정에 필요한 정책 문서가 전부 공개.
- **플러그인 방향 — `취소·환불 규정 네비게이터`**
  - 입력: 예약 유형(항공/숙박/투어)·결제 시점·취소 사유
  - 처리: 조건 분기(24시간 이내 여부, 부득이한 사유 여부 등)로 적용 규정·필요 증빙·예상 소요일·수수료 여부를 공식 정책 문서에서 결정
  - 출력: 적용 규정 요약·필요 서류 체크리스트·예상 처리기간·근거 링크
  - 형태: SKILL + 정책 규칙(공개 문서 기반 결정 트리)

### 문제 3-C · 공개 도움말 콘텐츠의 정합성·최신성 결함 (구조 개선·제안형)
- **근거(공개)**: 24시간 무료취소 정책 문서가 실존하나 **본문 텍스트가 사실상 비어 있음(이미지·첨부 위주)**을 직접 확인 `[검증됨]`. 즉 공개 도움말 자체에 빈 문서·최신성 불명 항목이 존재.
- **공개자료만으로 가능한 이유**: 점검 대상이 공개 도움말 그 자체. 외부 데이터가 전혀 필요 없음.
- **플러그인 방향 — `도움말 정합성·최신성 감사기`**
  - 입력: 공식 도움말 문서 모음(URL/텍스트)
  - 처리: ① 빈 문서·텍스트 부재(접근성/검색 불가) 검출 ② 문서 간 상호 모순(예: 취소 소요일 표기 불일치) 탐지 ③ 오래된 정책·날짜 미표기 항목 식별 ④ 카테고리 커버리지 공백 도출
  - 출력: 결함 목록·중요도·개선 제안(우선순위)
  - 형태: SKILL + 문서 파싱/비교 코드. "구조 개선·제안"에 직접 부합하며, 심사자가 공개 도움말로 즉시 재현 가능
